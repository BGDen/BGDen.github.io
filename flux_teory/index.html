Ответы:

Flux — это вид архитектуры, которую Facebook использует, чтобы работать с React.
Flux, шаблон проектирования, соответствует M в MVC.
Flux больше чем шаблон, больше чем фреймворк и имеет 4 главных компонента:

  • Действие (Action) — помощники, которые передают данные в Dispatcher
  • Диспетчер (Dispatcher) — получает эти действия и передает полезную нагрузку зарегистрированным callback-ом.
  • Хранилище (Stores) — действуют как контейнеры для состояния приложения и логики. Реальная работа приложения происходит в Stores. Stores, зарегистрированные для прослушивания действий Dispatcher, будут соответственно и обновлять View.
  • Представления (Views) — React компоненты захватывают состояние из Stores, а затем передают дочерним компонентам.

Основная проблема с MVC заключается в том, что он недостаточно масштабируется для огромной базы кода Facebook. Flux доказал свою эффективность, потому что все, что связано с Flux, касается тонкой настройки потока внутри приложения.
Архитектура MVC сияет на бэкэнде, где вся клиентская сторона может рассматриваться как представление. Это в отличие от Flux, который в значительной степени является интерфейсным шаблоном, который решает проблему управления состоянием приложения на стороне клиента.

    MVC

  • Двунаправленный поток данных (Bidirectional)
  • Привязка Data является ключом
  • Controllers обрабатывают бизнес-логику
  • Несколько синхронный (synchronous)
  • Надежная архитектура, используемая многими компаниями во многих проектах

    Flux

  • Однонаправленный поток данных (Unidirectional)
  • Events или Actions являются основными игроками
  • Store делает все расчеты
  • Может быть реализован как полностью асинхронный (asynchronous)
  • Помогает решать сложные проблемы, которые MVC не мог

Отличия:
Во Flux-архитектуре функцию Observer (Publisher) выполняет Диспетчер, где хранилища подписываются на связанные с ними действия и обрабатывают их;
Callback-и не привязаны к конкретным событиям. Каждая полезная нагрузка будет отправлена на каждый зарегистрированный вызов;
Их возможно частично или полностью отложить до тех пор, пока не будут выполнены все другие обратные вызовы;

Redux – это библиотека, предназначенная для управления состоянием JavaScript приложения. Чаще всего используется в связке с React или Angular для разработки клиентской части. Содержит ряд инструментов, позволяющих значительно упростить передачу данных хранилища через контекст.
